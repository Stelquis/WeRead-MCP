# WeRead-MCP 全面评估报告

> 评估日期：2026-07-17
> 版本：v1.0.0

---

## 一、架构评估

| 维度 | 评分 | 评语 |
|------|------|------|
| **模块划分** | ⭐⭐⭐⭐⭐ | 5 个模块职责清晰：main(入口)→server(协议)→scraper(HTTP)→parser(HTML解析)→error(错误)，单向依赖，无循环引用 |
| **关注点分离** | ⭐⭐⭐⭐⭐ | IO 密集型（scraper）与 CPU 密集型（parser）分离，异步边界清晰 |
| **扩展性** | ⭐⭐⭐⭐ | 新增工具只需在 server.rs 加 `#[tool]` 方法；新增数据源只需扩展 scraper |
| **MCP 协议合规** | ⭐⭐⭐⭐ | 严格遵循 stdio JSON-RPC 2.0，兼容 rmcp 官方 SDK，但工具返回纯 String 而非结构化 Content 数组，非最规范用法 |

### 亮点

纯 HTTP 替代 Headless Chrome 是架构层面的正确决策——二进制从 ~300MB 降到 ~20MB，启动从秒级降到毫秒级。

### 可改进

工具返回类型用 `String` 而非 `Vec<Content>`，rmcp 的 `#[tool]` 宏支持返回 `Result<impl Into<Vec<Content>>>`，可以返回更丰富的结构化结果。

## 二、代码质量

| 维度 | 评分 | 评语 |
|------|------|------|
| **类型安全** | ⭐⭐⭐⭐⭐ | Rust 语言强保障，thiserror 统一错误类型，serde 序列化 |
| **错误处理** | ⭐⭐⭐⭐⭐ | 所有 IO 操作都有 `map_err` 转换，网络请求有重试 + 指数退避 |
| **测试覆盖** | ⭐⭐⭐ | 13 个单元测试，覆盖 parser 解析、scraper 文件名生成、server 文件名清理。缺少集成测试（mock HTTP 服务器测试完整爬取流程） |
| **文档质量** | ⭐⭐⭐⭐⭐ | 中英文双语 README，每个函数都有 doc comment，CHANGELOG 维护规范 |
| **CI/CD** | ⭐⭐⭐⭐⭐ | fmt → clippy → build → test 全链路，工具链版本锁定 1.97.0 |

### 亮点

代码中无 TODO/FIXME 残留，所有 unwrap 都有正当理由（Selector::parse 在编译期可知正确性），error 变体设计合理。

### 可改进

缺少集成测试，无法在 CI 中验证真实微信 URL 的解析效果。

## 三、功能完整性

| 功能 | 评分 | 评语 |
|------|------|------|
| **文章爬取** | ⭐⭐⭐⭐⭐ | HTTP 请求 + 重试 + 缓存，稳定可靠 |
| **Markdown 转换** | ⭐⭐⭐⭐⭐ | 标题/列表/引用/代码块/表格/公式/图片全部支持，180 行转换逻辑覆盖主流标签 |
| **图片下载** | ⭐⭐⭐⭐ | 并发下载 + 信号量限流 + 重试，但缺少磁盘空间检查和图片大小限制 |
| **文件输出** | ⭐⭐⭐⭐ | 自动归档 + 路径替换，但中文目录名在 Windows 上可能有兼容问题 |
| **URL 校验** | ⭐⭐⭐⭐⭐ | 严格校验 `https://mp.weixin.qq.com/s/` 前缀，防误用 |
| **内存缓存** | ⭐⭐⭐⭐ | 50 条上限 + 30 分钟 TTL，但缺少 TTL 可配置化 |

### 亮点

图片扩展名推断逻辑（wx_fmt → URL 路径 → 默认 jpg）是实战打磨的结果，覆盖了微信图片 URL 的各种变体。

### 可改进

- 缺少对 `mp.weixin.qq.com/cgi-bin/` 等其他 URL 格式的支持
- 缺少请求频率限制（防反爬）

| **测试体验** | ⭐⭐⭐⭐⭐ | `test_mcp.py` 脚本开箱即用，支持 `--binary`/`--output-dir` 参数 |
| **调试体验** | ⭐⭐⭐⭐ | tracing 日志输出到 stderr，不污染 MCP 的 stdout，但日志级别固定为 INFO，缺少 `--verbose` 参数 |
| **协议调试** | ⭐⭐⭐ | 没有提供 MCP 协议交互的原始日志转储，调试协议问题时需要手动抓 stdout |

## 五、场景覆盖

| 场景 | 支持 | 说明 |
|------|------|------|
| **Claude Desktop** | ✅ 完全支持 | MCP 标准协议 |
| **Claude Code / Codex** | ✅ 完全支持 | `.mcp.json` 配置 |
| **Cursor** | ✅ 完全支持 | 设置中配置 |
| **OpenCode** | ✅ 完全支持 | `.mcp.json` 配置 |
| **Python 脚本直接调用** | ✅ 完全支持 | `test_mcp.py` 提供 |
| **批量爬取多篇文章** | ❌ 不支持 | 一次只能处理一个 URL |
| **自定义输出模板** | ❌ 不支持 | 输出格式固定为 article.md + images/ |
| **断点续传/失败重入** | ❌ 不支持 | 爬取中断后只能重新开始 |

## 七、风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 微信 HTML 结构变更 | 中 | 高 | 多选择器兜底机制已部分缓解，但需持续监控 |
| 微信反爬升级 | 中 | 中 | 可增加 IP 代理池、随机延迟、Cookie 管理 |
| 图片 CDN 限流 | 低 | 低 | 并发 5 张限制 + 重试 2 次已缓解 |
| 中文文件名跨平台 | 低 | 低 | sanitize 已处理非法字符，但 emoji 等未处理 |
